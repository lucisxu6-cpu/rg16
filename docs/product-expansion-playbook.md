# RG16 内容增长与复用扩展手册

## 1. 产品北极星

不是“测得准”与“传播强”二选一，而是同时成立：

1. 测得准：题目科学、评分透明、结论可追溯。
2. 传播强：结果具备身份表达价值，用户愿意主动分享。

## 2. 选题标准（决定是否立项）

一个测评只有同时满足以下条件才建议上线：

1. 有身份表达价值：用户愿意把结果发给朋友。
2. 有社交讨论空间：结果能引发“你真的是这样吗”的互动。
3. 有行动价值：结果可以转化为明确建议，而不是止于标签。
4. 可做证据链：关键结论必须能回溯到题目选择。

## 3. 推荐选题优先级

### P0（最快形成裂变）

1. 关系沟通风格（亲密关系 + 日常冲突）
2. 职场影响力画像（推进、协作、说服）
3. 社交角色原型（群体中的自然定位）

### P1（提高付费深度）

1. 压力恢复系统（压力触发器、恢复路径）
2. 目标执行模式（计划-执行-复盘风格）
3. 学习策略偏好（输入、理解、输出）

## 4. 每个测评的统一产品模板

1. 免费结果：
   - 一个主结论（适合分享）
   - 三条现实映射（工作/关系/压力）
   - 三条行动建议（可立刻执行）
2. 付费结果：
   - 逐维深度解读
   - 关键题证据链
   - 低/中/高层级与升阶建议
3. 分享产物：
   - 标准化分享短文案
   - 可视化结果卡片（后续可接生成图）

## 5. 研发复用规范

新增测评时，优先遵守如下结构：

1. `src/lib/assessments/types.ts`：通用输入/输出类型。
2. `src/lib/assessments/definitions.ts`：注册 parse + compute 逻辑。
3. `src/app/api/assessments/route.ts`：只做统一分发和持久化。

目标：新增测评只改“定义层”，不改“主链路”。
